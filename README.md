Сучасна масштабована веб-платформа для управління заходами та автоматизації продажів. Головна "фішка" проекту — мікросервісна архітектура та інтегрований алгоритм динамічного ціноутворення (Urgency-based Pricing), що імітує роботу ШІ для максимізації прибутку організаторів.

## 🚀 Основні можливості

-   **Система ролей (RBAC):**
    -   👤 **Покупець:** Каталог подій, пошук, пагінація, купівля та перегляд особистих квитків.
    -   🏢 **Організатор:** Створення івентів та аналітичний дашборд (конверсія, дохід).
    -   🛡️ **Адміністратор:** Повна модерація контенту та керування користувачами.
-   **⚡ Динамічне ціноутворення:** Автоматичний перерахунок вартості квитка через мікросервіс на Python.
-   **🔍 Смарт-фільтрація:** Пошук у реальному часі (з Debounce) та сортування за датою/ціною.
-   **🌓 Темна тема:** Адаптивний інтерфейс для комфортної роботи 24/7.

## 🏗️ Архітектура системи

Проект побудований на принципах слабкої зв'язності (loose coupling):

1.  **Frontend (Vite + React):** SPA-додаток з миттєвим оновленням UI без перезавантаження сторінки.
2.  **API Gateway (Node.js + Express):** Центральний вузол, що обробляє бізнес-логіку, авторизацію (JWT) та транзакції в БД.
3.  **Pricing Engine (Python + FastAPI):** Ізольований мікросервіс для складних математичних розрахунків.
4.  **Database (SQLite):** Реляційне сховище для забезпечення цілісності фінансових даних.

## 🧠 Алгоритм ціноутворення

Логіка мікросервісу на Python базується на двох факторах:
-   **Фактор дефіциту:** Ціна автоматично зростає на **10% від базової** за кожні продані **10% квитків** від загальної місткості залу.
-   **Фактор часу (Urgency Tax):** Якщо до початку події залишається **5 днів або менше**, алгоритм накладає додатковий мультиплікатор **1.2x (+20%)** до вартості.

## 🛠️ Встановлення та запуск

### Вимоги:
- Node.js v18.x+
- Python 3.10+
- npm & pip

### Швидкий старт (Windows):
Клонуйте репозиторій та просто запустіть скрипт ініціалізації:

```bash
# 1. Клонувати проект
git clone [https://github.com/ZENQU/ticket_sell.git](https://github.com/ZENQU/ticket_sell.git)
cd ticket_sell

# 2. Встановити залежності для кожного модуля (виконується один раз)
cd frontend && npm install
cd ../api-gateway && npm install
cd ../pricing-engine && pip install fastapi uvicorn pydantic

# 3. Запустити все одним кліком
cd ..
startkr.bat

📊 Схема даних

База даних містить 5 основних таблиць:

    users — автентифікація (bcrypt) та профілі.

    events — параметри подій та ліміти.

    orders — логування намірів покупки.

    payments — фінансові транзакції.

    tickets — згенеровані електронні квитки.
